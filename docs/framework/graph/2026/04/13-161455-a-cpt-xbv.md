Completed a seven-angle research effort on SDD distribution tooling to inform the decision for s-cpt-4gj. Angles: (1) prior art for Go CLI + Claude Code skills combos, (2) Go CLI distribution best practices in 2026, (3) macOS code signing/notarization for open-source Go binaries, (4) current state of the Claude Code plugin ecosystem, (5) bootstrapping patterns for two-stage installs (rustup, uv, gh, mise), (6) strategic bet between native Claude Code marketplace vs. conventional package managers, (7) version skew patterns between binary and companion skills (Helm, Terraform, asdf). Research used perplexity_research per angle; findings informed the subsequent distribution strategy decision. Key convergences: conventional package-manager distribution (GoReleaser + Homebrew tap) is what every comparable Go CLI landed on; two-stage install (brew install X then X setup) is the idiomatic pattern; bundling skills in the binary is the most robust answer to version skew; notarization is not blocking for Homebrew-delivered binaries; plugin marketplace is a real but niche/evolving secondary channel. Session-introduced finding beyond the research: binstaller (github.com/binary-install/binstaller) is the purpose-built companion to GoReleaser for generating an install.sh with embedded checksums and GitHub attestation chain-of-trust — chosen over hand-written.
