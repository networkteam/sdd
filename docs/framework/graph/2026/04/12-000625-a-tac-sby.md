Implemented pre-flight validator per plan d-tac-hjp. (1) Prompt templates: 7 files in sdd/preflight_templates/ — 6 check-type templates (closing_action, closing_decision, decision_refs, action_closes_signals, signal_capture, supersedes) plus contracts.tmpl partial included via template directive. Embedded via go:embed. Templates use text/template with structured context variables. (2) Pre-flight library in sdd/preflight.go: PreflightRunner interface, CheckType enum with String(), SelectCheckType pure function, AssembleContext gathering refs/closes/supersedes/contracts/plan-items/open-signals, FormatEntryForPrompt, RenderPrompt parsing all templates together for partial resolution, ParseResult handling PASS/FAIL/malformed output, RunPreflight orchestrator. No os/exec imports, all dependencies injected. Deviation: no PreflightFailedError type — during design dialogue we decided the library returns PreflightResult with Pass/Gaps and errors only for infra failures; the CLI handles fail results directly. This is simpler and the typed error added no value since the CLI is the only consumer. (3) CLI integration in cmd/sdd/main.go: claudeRunner type calling claude -p, --skip-preflight flag (adds preflight:skipped annotation), --preflight-model flag (default claude-haiku-4-5-20251001), wired between ValidateEntry and file write. (4) Graceful degradation: 30s context timeout on runner, claude not found or timeout produces warning + preflight:error annotation, validation FAIL rejects entry with gap list. (5) Unit tests in sdd/preflight_test.go: TestSelectCheckType (12 cases), TestCheckTypeString, TestFormatEntryForPrompt (3 variants), TestAssembleContext (7 variants covering refs/closes/contracts/supersedes/open-signals), TestRenderPrompt (all check types + invalid + conditional sections), TestParseResult (11 cases), TestRunPreflight (pass/fail/runner-error/parse-error/check-type-selection). All deterministic with mock runner. Deviation: no TestPreflightFailedError since the type was not implemented per item 2 deviation. (6) Evaluation tests in sdd/preflight_eval_test.go behind //go:build eval: 6 golden cases — missing plan items (true positive), full coverage with different wording (true negative), smuggled decision signal (true positive), valid signal (true negative), real graph history a-tac-tsd silent scope-out (true positive), contract violation (true positive). All 6 pass. Reports via t.Log/t.Errorf.
