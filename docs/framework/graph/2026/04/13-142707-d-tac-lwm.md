Migrate framework Go code to CQRS per d-cpt-l3s and the planning-discipline contract d-cpt-ah1. Single coherent scope; first refactor reviewed with user, rest mechanical.

**Sequence**:
1. **Set up packages**: create \`framework/sdd/command/\`, \`query/\`, \`handlers/\`, \`finders/\`, \`model/\`. Move pure model types from existing \`sdd/\` (Entry, Layer, Type, Kind, parsing helpers, validation logic) into \`model/\`. IO functions (\`LoadGraph\`, etc.) stay in \`sdd/\` as primitives until absorbed into finders during step 6.
2. **Refactor \`sdd new\` as the reference implementation**:
   - \`command.NewEntryCmd\` with all current flag fields + \`OnNewEntry func(id string)\` callback (just the new entry's identifier — full path or other detail retrievable via finder if the CLI needs it). \`Validate()\` checks required fields.
   - \`query.PreflightQuery\` + \`finders.Preflight(...)\` for the gate.
   - \`handlers.Handler.NewEntry(ctx, cmd)\` doing: validate command, load graph, validate entry, run pre-flight via finder, save stdin on rejection/dry-run, write entry file, copy attachments, commit, fire callback with the new ID.
   - Inject dependencies (graph dir, git committer, pre-flight runner, stdin reader, clock).
   - Update \`cmd/sdd/main.go newCmd()\` to be a thin shell: parse flags into \`NewEntryCmd\`, wire the callback for stdout output (using a finder to fetch the path from the ID if needed), instantiate \`Handler\` with production deps, call \`.NewEntry(ctx, cmd)\`.
3. **Review point with user.** Confirm shape before mechanical migration.
4. **Add the dry-run regression test** against \`Handler.NewEntry\` directly (honors d-prc-31v): build \`NewEntryCmd\` with \`--dry-run\` + invalid refs + stdin attachment, instantiate handler with temp graph dir + fake pre-flight runner, assert error returned and \`.sdd-tmp/\` contains the saved stdin.
5. **Migrate remaining commands**: \`lint\`, \`wip start\`, \`wip done\` → \`XxxCmd\` + \`Handler.Xxx\` + thin CLI shell.
6. **Migrate queries**: \`status\`, \`show\`, \`list\`, \`wip list\` → \`XxxQuery\` + finder + thin CLI shell.
7. **Update tests**: existing \`sdd/\` tests stay if they cover library primitives, move to \`finders/\` if they're pure-read tests.

**What stays the same**: on-disk graph format, CLI flags and command names, output formatting. This is internal restructuring for testability, not a UX change.

**What this enables**: d-prc-31v's first concrete application (the dry-run regression test). After this lands, "Action closure tightly bound to urfave/cli" is no longer a valid excuse — handler tests become the default unit of test for new command/query behavior.
