Keep the name 'SDD' — Signal-Dialogue-Decision names the core loop (Signal → Dialogue → Decision → Action → Signal...) that has been the framework's operating model from the start. d-cpt-omm reinforces this by collapsing the captured entry types down to Signal and Decision (with kinds), making the graph schema line up with the loop's named components. Graiki, Continuum, and Flow were branding explorations entertained while the framework's identity was still in flux; with the loop holding steady and the two-type redesign tightening its shape, no invented name is needed. 'SDD' maps 1:1 to the concept and works fine as both project and CLI name.
