Pre-flight validation exhibits find-something bias — the binary PASS/FAIL format incentivizes the LLM to stretch for gaps when none exist, producing false rejections (e.g. demanding CQRS decomposition for a logging plan, demanding neutrality justification for a dialogued design choice). A confidence-scored output format (high/medium/low per finding, string not numeric) would let the LLM name observations without forcing a verdict, with tooling applying a threshold to decide what blocks. The prompt must include calibration examples anchoring each level — e.g. high: type/layer mismatch, missing required refs; medium: potentially missing context that could be intentional; low: subjective preference about plan detail or style. Three levels are enough for the LLM to consistently assign; numeric scales invite clustering and calibration drift.
