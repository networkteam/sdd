Hierarchical output for sdd show multi-entry display. Primary entries (requested IDs) render as ## headers. Their refs nest with increasing heading depth (###, ####, etc.) following the transitive RefChain. Each entry prints full content on first appearance. Deduped entries print only the heading plus a '(shown above)' line. Blank line after every heading for consistent formatting. --- separates primary groups. For --downstream mode, the target entry is the primary, downstream entries are ### (flat — no transitive nesting since downstream doesn't recurse).

Implementation steps:
1. Refactor the show loop in main.go:167-201 — split RefChain result into primary (last) and refs (rest). Walk refs recursively to determine depth.
2. New printEntryHierarchical helper that takes a depth parameter and prints the appropriate heading level.
3. Downstream mode: primary is first entry, results are flat at ###.
4. Single-entry show gets the same treatment (primary ## + nested refs).
5. Add test coverage for multi-entry hierarchical output, dedup across primaries, and deep chains.
6. Rebuild binary (not committed).
