The agent consistently captures graph entries as compressed summaries without attachments, even when the conversation contains rich context (reasoning chains, comparisons, rejected alternatives, design dialogue) that would be valuable for future traversal. The human has to intervene to request attachments. Observed multiple times: in this session the agent proposed capturing the MemPalace analysis and the pre-flight validator design without attachments until explicitly prompted both times. Attachments should be the default expectation for entries emerging from substantive dialogue, not a special case the human must remember to request. Another instance of the pattern in s-stg-3vr — the agent knows attachments exist, has seen the pattern, but does not apply it without prompting.
