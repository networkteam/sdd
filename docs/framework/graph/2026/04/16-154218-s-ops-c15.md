CLI duration flags like --preflight-timeout accept raw integers (seconds) instead of Go-style duration strings (e.g. 180s, 2m30s). The flag errored on '180s' with a strconv.ParseInt parse failure. Duration flags should use time.ParseDuration for human-friendly input across all duration parameters, not just preflight-timeout.
