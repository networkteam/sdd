SDD graph lacks a visual browsing frontend. An sdd mirror command could generate an Obsidian-compatible copy of the graph — converting entry ID references into wikilinks in frontmatter for refs/closes/supersedes. The mirror is a local, uncommitted, read-only view — the canonical graph format stays tool-neutral. Research confirms: path derivation from entry ID is deterministic, Obsidian frontmatter and attachment links work natively. Core Obsidian graph view renders all links as undifferentiated edges; the Breadcrumbs community plugin can render frontmatter fields as distinct typed edges, but the mirror is useful even without it (clickable navigation, backlinks).
