Signal depth risk is addressed by the two-type redesign (d-cpt-omm) combined with pre-flight validation. Explicit signal kinds (gap, fact, question, insight, done) make shallow signals structurally visible — a feature request without a Why is a fact, not a gap, and pre-flight can calibrate validation differently per kind. This is a stronger solution than the original proposals (dialogue challenges alone or blanket pre-flight enforcement) because each kind gets appropriate validation depth: gaps require Why-level reasoning, facts can be lighter. Implementation of kinds is tracked by d-cpt-omm itself.
