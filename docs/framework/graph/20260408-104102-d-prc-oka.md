Grooming is a sub-skill (sdd-groom) that gathers context and a mode in /sdd that drives dialogue over the results. The sub-skill checks open signals and active decisions for: (1) downstream graph entries that look like resolution but lack closes fields, (2) recent Git activity (commit messages, changed files) that appears to address the entry but was never captured as an action. It returns candidates for closure with evidence. The /sdd skill drives the dialogue — walking through candidates, confirming closures, capturing missing actions. Additionally, the /sdd skill proactively suggests grooming when catch-up or status reveals many older open entries without downstream activity.
