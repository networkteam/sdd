This plan adds three diagnostic-and-correction tooling surfaces for stored summaries: an explicit-text override on `sdd summarize` so an outer agent or human can supply a better summary directly (bypassing LLM generation), summary text in the `sdd new` output so agents can verify fidelity at capture time before others read it, and a summary section in `sdd show` output that gives readers a quick orientation when looking up an ID and surfaces drift between body and summary as a side benefit during review. SKILL.md guidance closes the loop by instructing agents to check captured summaries against the body and offer correction. Closes the show-display gap (s-tac-dc3) directly and operationalizes one of the four candidate directions in the broader summary-drift gap (s-cpt-tdp) — explicit summary control via tooling, with capture-time fidelity check as the workflow that makes the override path actually used. The other three candidates (prompt hardening, scope definition, lint/diff tooling) remain open for separate dialogue. The explicit-text path is single-entry only — `--text` is rejected with `--all` or multiple IDs since manual override doesn't make sense in batch — and recomputes the prompt hash on write so subsequent automatic regenerations skip by hash unless `--force` is passed, mirroring LLM-generated behaviour. Decomposes across CQRS layers as: extend `SummarizeCmd` with explicit-text intent (command), branch in `Handler.Summarize` to skip the LLM call (handler), surface the generated summary through `NewEntryCmd`'s callback (command), no model or finder changes, and extend `WriteEntryFull` to render the summary block (presenter).

## Acceptance criteria

- [ ] `sdd summarize <id> --text <value>` writes the supplied text as the entry's summary, bypassing the LLM call. `<value>` may be a literal string or `-` to read from stdin.
- [ ] `--text` is rejected when `--all` is set or more than one entry ID is supplied; the error message names the constraint.
- [ ] Explicit-text summarize recomputes the summary hash from the current prompt input so subsequent automatic regenerations skip by hash unless `--force` is passed, matching LLM-generated behaviour.
- [ ] Explicit-text summarize commits the file with a message indicating the summary was supplied manually (not LLM-generated).
- [ ] `sdd new` output includes the generated summary text after the entry path, so agents can verify fidelity at capture time.
- [ ] `sdd show <id>` displays the stored summary as a labeled section between the frontmatter block and the body content; the section is omitted entirely when the entry has no stored summary.
- [ ] Existing `sdd show` behaviour (upstream/downstream summary lines, `--downstream`, `--max-depth`, dedup) is preserved.
- [ ] SKILL.md for `/sdd` instructs agents to verify the captured summary reflects the body's meaning after `sdd new` succeeds, and offer `sdd summarize <id> --text <value>` to correct it if it drifts.
- [ ] Unit tests cover: explicit-text writes summary and recomputes hash; rejection of `--text` with `--all` or multi-entry; show renders the summary section when stored summary is non-empty and omits it when empty.
- [ ] CLI reference under `internal/bundledskills/claude/` reflects the new `--text` flag on `sdd summarize`, the summary in `sdd new` output, and the summary section in `sdd show`.
- [ ] `go vet ./...`, `go test ./...`, and `golangci-lint run ./...` are clean.
