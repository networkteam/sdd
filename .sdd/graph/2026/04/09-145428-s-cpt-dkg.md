How does participant identity flow from session context into CLI commands? Currently the SDD skill infers participants from conversation context (git user, who's talking) and constructs --participant flags manually. No formal mechanism exists for the CLI to know who's invoking it. Surfaced during WIP marker implementation when agent defaulted to git config user.name without a decision. May matter more as concurrent multi-agent sessions become common.
