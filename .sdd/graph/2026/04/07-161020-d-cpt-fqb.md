Decisions have an optional kind property: directive (default, omitted) or contract. Directives request action and are closed when fulfilled. Contracts define standing constraints — they're never closed, only superseded. A directive can become a contract via a new decision with supersedes + kind: contract. Status view separates contracts from active directives. The sdd list command supports --kind contract|directive filtering.
