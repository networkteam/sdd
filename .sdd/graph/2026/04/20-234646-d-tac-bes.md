Introduce a pluggable LLM provider layer so SDD can swap between `claude -p` (default) and gollm-backed providers (Ollama, Anthropic API, OpenAI, etc.) via local config, and speed up `sdd summarize --all` through Anthropic prompt caching and errgroup-pooled concurrency.

Keeps `claude -p` as the default — no API key, piggybacks on the logged-in Claude Code session — a real onboarding advantage. gollm becomes the opt-in adapter for everything else. Extends the `config.local.yaml` schema established by d-tac-q5p with an `llm:` block; no new config infrastructure needed.

## Package layout

- `internal/llm/` — unchanged orchestration: templates, prompt assembly, JSON parsing, hash skip.
- `internal/llm/claude/` — existing shell bridge, stays.
- `internal/llm/gollm/` — new adapter wrapping `gollm.LLM.Generate` (plus `NewLLMWithMemory` when provider is Anthropic, enabling ephemeral system-prompt caching).
- `internal/llm/factory.go` — resolves Runner from config + flags.

## CQRS decomposition

This plan modifies internals of the `internal/llm/` dependency layer — it does not add new CQRS components. The existing decomposition is preserved:

- **Model**: `model.Config` gains an `LLMConfig` sub-struct (Provider, Model, Timeout, Concurrency) plus a pointer-typed `LLMConfigOverlay` for local overrides (APIKeys, OllamaEndpoint, RateLimitRPS, and overrides of any base field). A pure `MergeConfig(base, overlay)` applies non-nil overlay fields — same struct, local wins.
- **Commands**: existing `SummarizeCmd` gains `Concurrency int` and optional `Provider`/`Model` override fields. `NewEntryCmd` gains optional `Provider`/`Model` override fields. No new command structs.
- **Queries**: unchanged.
- **Handlers**: `SummarizeHandler.Summarize` changes its internal loop to an errgroup with `SetLimit(N)` and a `rate.Limiter` for remote providers. No new handler methods.
- **Finders**: unchanged surface; the `preflightRunner` they hold is now constructed via the factory.
- **CLI actions** (`cmd/sdd/`): `sdd new` and `sdd summarize` gain `--provider`/`--model` flags; `sdd summarize` gains `--concurrency`. `main.go` loads shared config then merges local overlay before wiring the factory.

The `Runner` interface change (`Request{SystemPrompt, UserPrompt}`) is an internal refactor of the dependency; callers in handlers and finders update their call sites but their responsibilities don't shift across CQRS layers.

## Runner interface change (load-bearing)

`Run(ctx, req Request)` where `Request = {SystemPrompt, UserPrompt}`.

- claude-cli: concatenates into single stdin blob (no caching path — unchanged behavior).
- gollm+anthropic: passes SystemPrompt via `AddStructuredMessage(..., "ephemeral")` with structured messages enabled; 5-minute server-side prefix cache yields 2–3× speedup from call #2 onward on batch summarize.
- gollm+ollama and gollm+openai: concatenates (no effective caching with the current gollm surface).

## Template refactor

Split `summary.tmpl` and all 8 pre-flight templates into a stable system partial (instructions, contracts, structural rules) plus a variable user partial (proposed entry, refs, closes). The stable partial is what gets cached.

## Concurrency (sdd summarize)

`golang.org/x/sync/errgroup` with `SetLimit(N)`; per-call context and timeout preserved. A `golang.org/x/time/rate.Limiter` gates remote providers only; Ollama and claude-cli run uncapped.

## Config shape

One `Config` struct, two files. Shared file loads into the base; local file is a pointer-typed overlay merged on top — non-nil overlay fields override, nil fields inherit. User-facing model: any field may appear in either file, local wins.

Committed defaults in `.sdd/config.yaml`:

```yaml
llm:
  provider: claude-cli
  model: claude-haiku-4-5-20251001
  timeout: 2m
  concurrency: 4
```

Local overrides in `.sdd/config.local.yaml` (gitignored, extends d-tac-q5p):

```yaml
llm:
  provider: ollama
  model: llama3.1:70b
  ollama_endpoint: http://localhost:11434
  api_keys:
    anthropic: sk-...
  rate_limit_rps: 4
```

Resolution: CLI flag > local > shared > built-in default.

## Design rationale

The Runner-interface split (System vs User) is what makes provider-agnostic caching effective. Anthropic prompt caching uses prefix matching; passing the stable template portion as a cacheable structured message (via gollm's ephemeral cache_control) guarantees the prefix is treated as cache-eligible regardless of how prompt rendering evolves. For providers without caching (Ollama, OpenAI via gollm), the split is free — the adapter concatenates and sends as one payload.

A client-side response-cache decorator was considered and dropped: pre-flight and summarize prompts are unique per entry, so a memoization layer would miss nearly every time. Anthropic Message Batches API was considered and dropped: orthogonal to interactive `sdd summarize` latency, and gollm has no wrapper to reuse.

See [plan](./20-234646-d-tac-bes/plan.md) for full alternatives considered and rejected paths.

## Acceptance criteria

- [ ] `llm.Runner.Run` takes `Request{SystemPrompt, UserPrompt}`; `RunResult` metadata unchanged
- [ ] `internal/llm/claude` concatenates System + User into a single stdin payload; existing behavior preserved
- [ ] `internal/llm/gollm` package implements `llm.Runner` via `github.com/teilomillet/gollm`
- [ ] gollm adapter enables structured messages + `ephemeral` cache control on SystemPrompt when `provider == "anthropic"`
- [ ] gollm adapter explicitly supports and is tested against `anthropic`, `openai`, `ollama`
- [ ] `summary.tmpl` split into stable system partial + variable user partial; rendered output semantically equivalent to the current template
- [ ] All 8 pre-flight templates split similarly; stable contracts and partials live in the system portion, proposed-entry and context in the user portion
- [ ] `internal/llm/factory.New(cfg)` constructs the correct Runner from resolved config
- [ ] `model.Config` gains an `LLMConfig` sub-struct; a pointer-typed `LLMConfigOverlay` supports local-file overrides; a pure `MergeConfig` function applies non-nil overlay fields over the base
- [ ] `.sdd/config.yaml` schema gains an `llm:` block (provider, model, timeout, concurrency) with documented defaults
- [ ] `.sdd/config.local.yaml` schema gains an `llm:` block supporting provider / model / api_keys / ollama_endpoint / rate_limit_rps overrides (extends d-tac-q5p, no new file infrastructure)
- [ ] Config resolution order implemented and tested: CLI flag > local > shared > built-in default
- [ ] `sdd summarize` gains `--concurrency N`; uses `errgroup.SetLimit(N)`; default from config
- [ ] Remote providers (anthropic, openai) gated by a shared `rate.Limiter`; Ollama and claude-cli uncapped
- [ ] `sdd summarize` and `sdd new` gain `--provider` and `--model` flags; existing `--preflight-model` kept as an alias for `--model` in pre-flight context
- [ ] Hash-based summary skip (`SummaryHash`) still works; the hash is computed on the combined System+User payload
- [ ] Error messages distinguish "provider not configured", "api key missing for provider X", and "provider unreachable"
- [ ] `go vet ./...`, `go test ./...`, `golangci-lint run ./...` clean
- [ ] New tests cover: factory resolution; claude adapter concatenation; gollm adapter happy path with a mocked provider; end-to-end with local Ollama behind a build tag (skipped in CI without an endpoint)
