Bound sdd show output with depth-limited BFS traversal using entry summaries. Target entry (depth 0) renders full content; all upstream (refs/closes/supersedes) and downstream (refd-by/closed-by/superseded-by) entries at depth 1+ render summary + ID + kind only. BFS traversal per direction with per-direction dedup — each entry shown at shallowest occurrence, cross-reference marker for later encounters. --max-depth flag with default 4, independent per direction (4 upstream, 4 downstream). Truncation marker at boundary when deeper entries exist. Output format per summary line: {relations} {full-id} ({kind}): {summary} — where relations are comma-joined when an entry has multiple edge types to its parent (e.g. refs,closes). Kind always shown in parentheses when present, future-proofed for two-type redesign (20260412-114647-d-cpt-omm) where every entry has a kind. Depends on entry summaries from 20260414-152337-d-tac-h93 being populated. See [plan](./14-155925-d-tac-017/plan.md) for implementation details and output examples.
