`sdd list` orders entries ascending (oldest first), so reading "recent activity" requires scrolling to the bottom or piping through `tail`. For agents, the common need is cheap access to "the last day" or "the last N entries" — the current shape forces post-processing. The design question is interface, not sort order: flags like `--since <duration>`, `--limit N`, or `--tail N` would let an agent ask for the tail slice directly. Worth exploring which shape fits — `--since 24h` is intuitive for time windows; `--limit N` works for fixed counts; both could coexist.
