Add background sync awareness to the SDD CLI and skill, with explicit conflict-safety prediction before any auto-rebase. On any sdd command except `sdd init`, auto-fetch when last-fetch exceeds a configurable cooldown (default 15 min from `.sdd/config.yaml`); compare local vs remote by scanning commit messages with the `^sdd:` prefix (CLI-controlled, fully covers entry-changing and graph-adjacent commits); report counts bidirectionally via `slog`. When both sides have diverged graph-commit history, the CLI consults `git merge-tree` to predict whether a rebase would be clean and surfaces the result as a distinct `slog` state (fast-forward, clean-rebase-predicted, conflict-predicted). The SDD skill auto-pulls `--rebase` only on fast-forward or clean-rebase-predicted states with a clean working tree, tell-then-run style; on conflict-predicted or dirty working tree it warns with paths and defers to the user. Missing git setup (no repo / no remote / no upstream) and transient fetch failures emit `slog.Warn` once per cooldown window and proceed. Supersedes `d-tac-n6y` by adding explicit conflict prediction (new AC6) and tying the skill's auto-pull contract to CLI-reported safety (refined AC5); all other ACs carry unchanged.

## Acceptance criteria
- [ ] sdd commands (except `sdd init`) run git fetch when last-fetch exceeds cooldown from config; default cooldown is 15 minutes; timestamp persisted in .sdd/tmp/last-fetch
- [ ] After fetch, CLI detects remote-ahead graph commits by `^sdd:` commit-message pattern and emits a slog line with count
- [ ] CLI detects local-ahead graph commits the same way and emits a slog line with count
- [ ] Git command count minimal: one fetch per cooldown window, log comparison via `git log` range, no file diffs
- [ ] CLI predicts rebase conflict safety when local and remote have diverged: consults `git merge-tree --write-tree --merge-base=<base> HEAD @{u}` and emits distinct slog states for fast-forward, clean-rebase-predicted, and conflict-predicted (with paths on conflict)
- [ ] SDD skill auto-pulls `--rebase` (tell-then-run) only on clean working tree + (fast-forward OR clean-rebase-predicted); on conflict-predicted or dirty working tree, warns with conflicting paths and defers to user; suggests push when local is ahead
- [ ] Missing git setup (not a repo / no remote / no upstream) and transient fetch failures emit `slog.Warn` at most once per cooldown window; commands proceed without error
