`/sdd-explore` returns raw `sdd show` output for the full upstream chain, downstream entries, and related entries, pushing all sense-making onto the outer agent and defeating the fork-compresses principle. The current "no summarization" rule (d-tac-vd8) was a deliberate choice to avoid losing context from rich entries — that concern remains valid and must be preserved. But dumping everything wastes both the fork's isolated context (spent reading) and the outer context (spent re-reading), and leaves the outer agent to do the prioritization work that a forked sub-skill exists to absorb.

Fix direction (following the d-ops-258 precedent for `sdd-catchup`): prioritize by grouping and detail level, not by omission. Summarize each chain (upstream / downstream / related) into a narrative briefing with entry IDs exposed; mark continuation points where detail was compressed ("N more ancestors truncated", "3 related entries omitted"); let the outer agent request specific raw entries on demand. Progressive disclosure instead of raw dump. Composes with the `sdd show --max-depth` direction from s-ops-dd8.
