# Key takeaways: responsibility/presence imbalance
Talk: https://www.youtube.com/watch?v=_Zcw_sVF6hU
Speakers: Armin Ronacher & Cristina Poncela Cubeiro (Earendil)

## Creating entities outpace responsible entities

> "the total number of entities both humans and machines that are participating in code creation process outnumbers the ones that can carry responsibility"

> "every engineer has a multitude of producing power compared to their reviewing power"

> "we are piling up on pull requests... more and more code reviews being skipped, being rubber stamped"

> "we're not there where the machine can be responsible for the code changes"

> "the responsibility still rests with the engineering team"

## Amplification without accountability

> "I have marketing people shipping code. I have former CEOs that used to be like engineers now shipping code again. And so the roles that those people have in the companies also doesn't give them... the responsibility doesn't rest in them."

> "this amplification is something that at the very least we need to recognize"

> "when you get this pull request that looks really daunting and has 5,000 lines of code in it, this is actually when you should be thinking, and that's exactly when it's the most overwhelming, and increasingly we're tapping out of this"
