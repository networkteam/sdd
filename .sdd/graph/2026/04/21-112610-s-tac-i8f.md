## Deviations / follow-ups surfaced during testing

- Post-plan commit 039e699 hardened the batch loop: plain errgroup (not WithContext) so one timeout no longer cancels siblings; gollm SetMaxRetries(0) to stop retry-WARN floods; SetMaxTokens(4096) to prevent mid-sentence truncation under Anthropic (gollm's default 256 silently truncated longer summaries); resolveTimeout helper so llm.timeout in config actually drives per-call context timeout. logCallResult now always emits op+duration (was bailing on nil Meta, which is gollm's normal path).
- gollm's Generate(...) returns text only — token/cost metadata is extracted internally and then dropped. Captured as part of s-cpt-ed2. Claude-cli path still surfaces full metrics.
- Cross-model summary quality surfaced template-level weaknesses (self-ID references, "This signal identifies..." openers) on both gemma4:26b and Haiku — captured in s-cpt-ed2 (supersedes s-cpt-nys). Not scoped to this plan.
- Observed intermittent "claude -p: exit status 1" on certain pre-flight prompts during testing via --provider claude-cli override. Did not reproduce on the default claude-cli path; root cause not determined; worth watching on future captures.
