# Key Takeaways — Verbatim

Source: Simon Willison, "An AI state of the union: We've passed the inflection point and dark factories are coming," Lenny's Podcast
Video: https://www.youtube.com/watch?v=wc8FBhQtdsA

## Tests and docs are no longer quality signals

> "it used to be if you looked at software and it had high quality tests and documentation, everything, it meant it was good. And now that signal is gone."

## Proof of usage is the new signal

> [Lenny]: "It's almost like we need a proof of work for this versus the—"
> [Simon]: "proof of usage. Proof of. Exactly."

## The problem with rushing past usage

> "I don't believe in it. And the reason I don't believe in it is that I got to rush through all of those things. I think the quality is probably good, but I haven't spent enough time with it to to feel confident in that quality. Most importantly, I haven't used it yet. Like, it turns out when I'm using somebody else's software, the thing I care most about is I want them to have used it for for months, right? I want other people to have put that software into practice."

> "I've got some very cool software that I built that I've never used. Like, it was so it was quicker to build it than to actually try and use it."
