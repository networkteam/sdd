Fixed drift between SDD reference docs and CLI behavior by adding `--kind plan` as a valid value, documenting `--skip-preflight`/`--dry-run`/`--preflight-timeout`, defining the `plan` kind in framework-concepts, and demonstrating the heredoc-to-variable pattern for multi-line descriptions in both SKILL.md and cli-reference.md (commits `b2748ac`, `26e8b8e`).

Gap discovered while capturing the fork plan (d-tac-v4f): the CLI accepts `plan` as a kind and exposes several pre-flight flags that SKILL.md references but cli-reference.md did not list. The heredoc pattern for long descriptions — write a `DESC=$(cat <<'EOF' ... EOF)` and pipe attachment content via stdin — was also undocumented, so agents default to temp files under `.sdd/tmp/` when a cleaner pattern exists. No prior signal tracked this; captured as a standalone process action to leave a marker future readers can trace.
