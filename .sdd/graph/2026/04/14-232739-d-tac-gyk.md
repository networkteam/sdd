Surface LLM call metadata through an agent-neutral abstraction in llm.Runner, per the agent-platform neutrality directive (d-stg-574) and explicit dialogue where the user identified that metadata must not be Claude-specific so other runners can populate the same struct. Change Runner.Run to return a result struct with response text and agent-neutral LLMMetadata. Move claudeRunner from cmd/sdd/main.go into llm/claude subpackage — claude-specific JSON parsing stays isolated, the llm package has no provider imports. The claude runner uses --output-format json, parses the full response, and maps it to the neutral metadata struct. Logging via log/slog with github.com/networkteam/slogutils for context-carried loggers — CLI command handlers set "command" attr, llm package emits one debug log call per LLM invocation under slog.Group("llm") with op, duration, and a sub-group per model containing tokens in/out and cost (when available). Internal helper called by Preflight and Summarize after Run returns. Verbosity tiers: default warn (no LLM output), -v info, -vv debug (LLM call details visible).
