SDD session entry points (sdd status, /sdd-catchup, /sdd-explore) consume excessive tokens and wall-clock time. The 209-entry graph requires full content parsing on every invocation with no pre-summarization — combined with unbounded ref expansion (s-ops-dd8) and raw output delegation to the outer agent (s-prc-6hk), every session start is expensive. Entry-embedded summaries — LLM-generated, direct-relationship-only, stitched into chains at read time — would bound output size and shift summarization intelligence to the consuming layer where read context is known.
