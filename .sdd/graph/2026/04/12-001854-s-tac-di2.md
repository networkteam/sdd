Evaluated pre-flight validator by using it to close its own implementation plan (d-tac-hjp). First run caught 5 legitimate gaps in plan coverage — all real. Discovered three design adjustments: (1) hard failure on all errors including infra, not graceful degradation — silent preflight:error annotation gives false confidence, user must explicitly --skip-preflight; (2) closing-action template needs to accept explicit deviations with reasoning, distinguishing conscious decisions from silent omissions; (3) timeout needs to be generous (120s default) and configurable for large prompts. All adjustments implemented and committed. 6/6 eval golden cases pass. The generation-validation separation principle from s-cpt-91k is validated in practice.
