Every bug fix requires a regression test in the same change. A bug fix is a change that corrects behavior the system should not have exhibited — wrong output, broken logic, unintended side effects. The test must fail before the fix and pass after. Test-first is advised when practical. If a fix is genuinely hard to test, treat that as a signal the code's structure should change, not a justification for skipping the test.

New features should include tests for core behavior, but coverage scope is a judgment call based on complexity and risk — not every helper or wiring change needs a test.

Not bug fixes (no regression test required): refinements to correct-but-imprecise behavior — template wording improvements, prompt tuning, definition sharpening, calibration adjustments. These are improvements, not corrections. The distinction: a bug fix addresses something that was wrong; a refinement makes something that worked become clearer or sharper.
