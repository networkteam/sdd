Landed the pre-flight template overhaul and retirement-path structural relaxation for session 2 of the two-type redesign (d-tac-o1k / d-cpt-e7i), renaming two templates, adding three new templates and two shared partials, extending two existing templates, broadening dispatch to kind-aware routing across 8 check types, and relaxing `validateCloses` so directive-kind decisions can retire contracts and aspirations. Implementation in commit e875905; `go vet`, `go test ./...`, and `golangci-lint run` all clean.

## AC coverage

- **AC #2 (templates per check transaction, not per kind)** — Templates stay one-per-transaction (closing_done, closing_decision, decision_refs, short_loop, dissolution, aspiration_capture, signal_capture, supersedes). Kind-awareness lives in dispatch (`selectCheckType`), not in template branches.
- **AC #3 (closing_action → closing_done)** — `git mv` rename; action references stripped throughout; framed around "completion entry (a `kind: done` signal)"; AC coverage calibration subsection preserved for plan closures.
- **AC #4 (action_closes_signals → short_loop)** — Renamed and retargeted for `kind: done` signal closing a `kind: gap` signal. Rewritten with the layer × scope-shape calibration matrix from the plan, plus gap-coverage and smuggled-decision checks. Each matrix cell carries a concrete example.
- **AC #5 (dissolution.tmpl)** — New template applied when fact or insight closes a question. Calibration tests dialogue-captured context presence (narrative references the question), never reasoning correctness. Three bands + enumerated "no finding" patterns.
- **AC #6 (unrelated_refs shared partial)** — New `{{ define "unrelated_refs" }}` partial guarded by `{{ if .ReferencedEntries }}`, wired into all seven capture templates (signal_capture, decision_refs, aspiration_capture, closing_decision, closing_done, short_loop, dissolution). Calibration flags topically-disconnected refs at high/medium/low, no-finding for genuine grounding.
- **AC #7 (unusual_close shared partial)** — New `{{ define "unusual_close" }}` partial wired into every close-carrying template (closing_decision, closing_done, short_loop, dissolution). Severity gated by rationale presence: high without rationale, low with rationale, no-finding for standard patterns (enumerated: decision→signal, done→decision, done→gap short-loop, fact/insight→question dissolution, directive→stable-kind retirement).
- **AC #8 (closing_decision retirement-rationale)** — Added a retirement-rationale calibration subsection for stable-kind targets (fact/insight/contract/aspiration) with four bands and concrete examples per band. **Implicit scope expansion:** relaxed `validateCloses` to allow `kind: directive` → `kind: contract|aspiration` closures (every other decision-closes-decision still rejected) and extended `Graph.Contracts()` / `Graph.Aspirations()` to also exclude closed entries. Without this, AC #8's retirement-rationale check was a dead branch for contract/aspiration — the check's structural gate is part of its applicability. Decision folded into session 2 after checking sessions 3–6 and confirming no other session naturally owns this (see dialogue note below).
- **AC #9 (decision_refs directive-reads-aspiration-shaped)** — Added subsection flagging `kind: directive` at stg/cpt with perpetual framing and no completion criterion. Three bands (medium / low / no-finding) with examples per the plan spec ("prioritizing user trust as an enduring stance" → medium; "direct-to-consumer over wholesale" → no finding).
- **AC #10 (aspiration_capture.tmpl)** — New template for `kind: aspiration` captures without closes. Gradient calibration — medium for un-acknowledged contradiction with an active aspiration, low for partial alignment, no-finding for reinforcement / orthogonal axis / acknowledged tension; explicit "never `high`" rule stated. Supporting model work: extended `preflightContext` with `ActiveAspirations`; added `Graph.Aspirations()` method and `Entry.IsAspiration()` helper (symmetric with existing `Contracts()` / `IsContract()`) to populate the context.
- **AC #11 (calibration discipline)** — Every new or modified band names a concrete observable pattern with at least one example; every template includes an explicit "no finding" enumeration. Verified band-by-band against the plan's calibration-discipline section.

## Dispatch overhaul

`selectCheckType` became kind-aware:
- Completion records (type=action legacy, or signal with kind=done) + closes → closing_done if any target is a decision, else short_loop
- Signal with kind=fact/insight + closes → dissolution (non-question targets flagged by unusual_close partial)
- Decision with kind=aspiration (any refs shape) → aspiration_capture
- Decision + closes → closing_decision
- Decision, no closes → decision_refs
- Signal otherwise → signal_capture
- Supersedes always takes priority
