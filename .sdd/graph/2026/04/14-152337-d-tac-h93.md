Embed LLM-generated summaries in entry frontmatter to replace recursive full-content expansion across all CLI output and skill invocations. Each summary describes the entry and its direct refs/closes/supersedes relationships only — transitive context emerges from chained summaries at read time. Generated at sdd new time from the in-memory graph, committed with the entry. A dedicated sdd summarize command regenerates summaries on demand for migration and maintenance. Skills consume summaries for closed entries and full details for open entries only. See [plan](./14-152337-d-tac-h93/plan.md) for implementation shape, prompt template design, CLI changes, and skill updates.
