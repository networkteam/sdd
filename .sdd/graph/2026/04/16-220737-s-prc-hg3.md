The `d-prc-8vh` contract requires regression tests for bug fixes, with a clear mechanism for code: pre-flight uses go tests under the `eval` build tag as persistent regression guards. Skill/prompt-level fixes — e.g., the `/sdd` Playbook rewrite captured in `a-prc-1nn` — have no equivalent; validation today is manual replay across fresh sessions, which leaves no artifact to catch future regressions. Open question, needs research: is automated regression testing of agent skills realistic (would require agent replay infrastructure, graph fixtures, stable eval criteria for narrative output), or is manual fresh-session replay the pragmatic ceiling — with `d-prc-8vh`'s rule applying only to code-path fixes?
