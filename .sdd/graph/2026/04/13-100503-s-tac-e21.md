Implemented stdin persistence and `--dry-run` flag on `sdd new` per d-tac-q5p. Both features landed together because they share the stdin-save logic.

**Stdin persistence**: `findStdinAttachment` and `saveStdinAttachment` helpers added. Stdin content is written to `<graph-dir>/.sdd-tmp/<hash>-<target>` where `<hash>` is the first 8 chars of sha256. A `reportSavedStdin` closure carries an idempotency flag (`stdinSaved`) so multiple invocations within the same command produce a single save and a single printout. The closure is called from the pre-flight rejection branch (real-run reject saves), from the post-pre-flight dry-run branch (dry-run pass saves), and from a `defer` registered when `--dry-run` is set (dry-run validation-failure saves — required because graph validation returns before the explicit branches are reached). Save failures emit warnings without masking the underlying validation or pre-flight result. Save is idempotent: same content + same target → same path, so re-runs don't accumulate files.

**No automatic cleanup**: per spec, `.sdd-tmp/` is never purged by `sdd new`. The agent removes files manually when iteration is done. This avoids parallel sessions wiping each other's saves and keeps the implementation small.

**`--dry-run` flag**: New bool flag in `newCmd`. After validation and pre-flight (or skip), if dry-run is set: stdin is saved (via the explicit success-branch call, with the defer as backstop), the function returns nil so the action exits 0, and no entry is written and no commit is made. The "non-zero on fail" half of the spec is satisfied by inheritance — graph validation and pre-flight rejection already return errors and urfave/cli maps any error returned from a command action to a non-zero exit. No new exit-code plumbing required; dry-run failures exit non-zero through the same path as real-run failures. Composes with `--skip-preflight` — structural validation still runs.

**Repo plumbing**: `docs/framework/graph/.gitignore` added with `.sdd-tmp/` to localize the rule next to the directory it governs.

**Tests**: unit tests for `findStdinAttachment` (returns stdin entry, nil otherwise) and `saveStdinAttachment` (path under `.sdd-tmp/`, 8-char hash prefix, content written, stable hash for same content, distinct hash for different content). The dry-run flow itself is exercised manually because the `Action` closure is tightly bound to urfave/cli plumbing; manual verification covered four cases — dry-run + skip-preflight + no stdin (exits 0, writes nothing), dry-run + skip-preflight + stdin pass (exits 0, saves once, prints retry hint), dry-run + skip-preflight + stdin + invalid ref (exits non-zero on validation failure, defer still saves stdin), and dry-run + skip-preflight + stdin pass (single save line — no double print from defer). Failure-exit behavior is inherited from existing error-handling code paths already covered by validation tests.
