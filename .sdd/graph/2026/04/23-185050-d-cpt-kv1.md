Operationalizes the priority synthesis from `s-cpt-w1m` by shaping a dedicated bootstrap sub-skill (`sdd-bootstrap`) that serves project adoption under the focus directive (`d-stg-u2i`). Matches the sub-skill pattern of `sdd-catchup`, `sdd-explore`, `sdd-groom` — scan state, structure findings, walk each finding with dialogue, capture on user approval.

The playbook comprises six adaptive moves: (1) readiness sweep via `sdd status` + `sdd lint`, classifying graph state; (2) brownfield project context gather reading README, `docs/`, `AGENTS.md` / `CLAUDE.md`, `git log`, and project structure, synthesizing a dialogue-grounding summary plus fact-capture and decision-mining candidates under a non-duplication rule; (3) actor capture loop with human-friendly prompts (not technical jargon — "how would you like to be named consistently here?" rather than "canonical?"), multi-participant support, and light role introduction deferred by default; (4) strategic/conceptual seeding via the Golden Circle — WHY→aspirations, HOW→strategic directives (contracts dropped; they emerge from working with the graph, not upfront), WHAT→directives — with 2–3 questions per pass and adaptive depth; (5) opportunistic role suggestion when patterns crystallize; (6) handoff via re-lint + readiness summary back to the `/sdd` main loop.

Four overarching principles: (a) read the user / match tempo — minimal ceremony by default, depth on engagement; (b) teaching mode by default with mode-switch on experience signal (brief framings explaining SDD vocabulary for first-timers, transparent `/sdd`-style captures for experienced users, orientation asks at the start); (c) brownfield before blank slate — gather context first, ground subsequent questions in it, fall back to greenfield only when context is sparse; (d) playback-before-capture preserved — bootstrap does not bypass the dialogue-first discipline.

Depends on the actors/roles type-system plan (`d-cpt-thg`) — the playbook captures actor and role entries that require those kinds to exist in the graph. Plan B lands first; Plan A's test-case AC (running on this repo to capture Christopher and Claude as actor signals, closing the current participant-coverage lint gap) cannot be validated before then. Plan A's scope is primarily skill bundle content (markdown), not Go code, so CQRS decomposition does not apply; any detection needed for auto-prompted invocation reuses Plan B's finders.

This repo is the first concrete test case: 353 entries with `participants` fields currently reference `Christopher` and `Claude` without matching actor signals. Running the playbook end-to-end on this graph validates all six moves, establishes the two actors with proper canonical + aliases + prose context, closes the participant-coverage lint findings, and proves the playbook works on real content. Greenfield validation comes through future adoption work under `d-stg-u2i`.

## Acceptance criteria

- [ ] Sub-skill bundle exists at `internal/bundledskills/claude/sdd-bootstrap/SKILL.md` with references as needed; extracted to `.claude/skills/sdd-bootstrap/` on `sdd init --scope project`
- [ ] Move 1 (readiness sweep): runs `sdd status` and `sdd lint`, classifies graph state as empty / populated-with-gaps / fully-bootstrapped, reports findings to the outer skill
- [ ] Move 2 (brownfield context gather): reads `README.md`, `docs/`, `AGENTS.md` / `CLAUDE.md` (any folder), recent `git log` entries, and project structure (top-level tree + manifest files); synthesizes a context summary presented to the user for confirmation; surfaces rough infrastructure facts (as `kind: fact` signal candidates, process layer) and decision-shaped claims from `AGENTS.md` / `CLAUDE.md` (as aspiration / directive / contract candidates) one-by-one with usual playback-and-confirm; applies the non-duplication rule (facts stay high-level and reference working docs rather than re-inlining). Skipped entirely for greenfield graphs.
- [ ] Move 3 (actor capture loop): local participant captured first with human-friendly prompts ("how would you like to be named consistently?", "other name variants from external sources?", "anything helpful about you in the project?"); multi-participant prompt ("are there others you want in the graph?") captures each additional actor; light role introduction after actors land with defer-by-default option
- [ ] Move 4 (strategic/conceptual seeding): Golden Circle pattern (WHY→aspirations, HOW→strategic directives, WHAT→directives) with contracts explicitly excluded from first-bootstrap; 2–3 questions per pass with adaptive depth (skip pass if nothing rises); questions grounded in brownfield context when present, blank-slate for greenfield
- [ ] Move 5 (role suggestion): opportunistic and deferred by default; offered only when contribution patterns are visible in actor captures or project context; easy to skip
- [ ] Move 6 (handoff): re-runs `sdd lint`, produces a readiness summary (N actors captured, M gaps closed, ready-or-not-ready assessment), returns the user to the `/sdd` main loop
- [ ] Teaching-mode preambles: first-time users see a 2–3 sentence framing before each capture's playback (what the entry type is, why it matters, what effect it has); experienced users (self-identified at orientation or inferred from fluent SDD vocabulary) get transparent `/sdd`-style captures without preambles
- [ ] Adaptive pacing honored: playbook advances on one crisp answer per pass; depth only on user engagement; no forced ceremony; fast path from orientation to handoff is possible in minutes for simple cases
- [ ] Invocation surfaces wired: explicit `/sdd-bootstrap`; auto-prompted by the main `/sdd` skill when lint reports unresolved participants or graph classifies as empty/sparse at catch-up time; main `/sdd` `SKILL.md` references the sub-skill as a playbook move with trigger conditions
- [ ] Playback-before-capture discipline preserved throughout: every capture within the playbook (facts, aspirations, directives, actors, roles, mined claims) uses the same play-back-and-confirm pattern as the `/sdd` main loop
- [ ] Test case on this repo (depends on `d-cpt-thg` landed): running `/sdd-bootstrap` captures `Christopher` and `Claude` as actor signals with proper canonical + aliases + prose context; `sdd lint` participant-coverage reports clean afterward; playbook handoff summary confirms readiness
