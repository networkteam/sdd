Enrich CLI `sdd status` and `sdd list` output with `kind` (decisions only for now; format accommodates future universality) and `participants` per entry; fix the `--participants` CSV whitespace-trim bug; normalize existing participant names in this repo to a consistent form as a one-time cleanup (repo-local hygiene, not a framework convention); update `/sdd-catchup` and the `/sdd` Catch-up Playbook to consume enriched output and surface participants narratively, not as per-item metadata.

Two signals converge. s-prc-omw surfaced participant-name inconsistency with four proposed fixes; this plan scopes fixes 1+2 (CSV trim + catch-up surfacing) and defers fixes 3+4 (whoami config, pre-flight validation). The new CLI-surface signal (s-prc-v69) names the broader gap: sub-skills can't read `kind` or `participants` from `status`/`list` without per-entry `sdd show`. Same emission change fixes both.

Format per line: `<full-id> <layer> <kind>? <type> [confidence: <conf>]? (<participants>) <summary>`. Kind only on decisions for now. Confidence on decisions + signals. Participants always present. Ordering puts graph position first, meta middle, participants + summary last.

Rationale for narrative-not-dashboard participants in catch-up: per-item attribution becomes noise when contributors are uniform (solo+AI, or repeating team). Surface only when actionable: thread-level mentions when the thread's contributors differentiate from the active recent set; optional header line; silent otherwise. CLI emits participants deterministically; rendering is the editorial layer.

Sub-skill role: `/sdd-catchup` stops per-entry `sdd show` for kind/participants after enrichment (reads status/list). It must forward kind and participants in canonical structured form to the outer agent, not only embedded in prose — otherwise the outer agent loses the facts for rendering. Thread clustering + narrative synthesis remain the sub-skill's irreducible value.

Alternatives deferred as separate signals: JSON output mode; refactor `/sdd-catchup` to return fully structured data; extend line format to refs/closes/supersedes; whoami config (s-prc-omw fix 3); pre-flight participant validation (s-prc-omw fix 4).

## Acceptance criteria

- [ ] `sdd status` emits per entry: `<full-id> <layer> <kind>? <type> [confidence: <conf>]? (<participants>) <summary>` — kind only for decisions, confidence only for decisions + signals, participants always
- [ ] `sdd list` emits the same per-entry format
- [ ] Grouping in `sdd status` (Contracts / Plans / Active Decisions / Open Signals / Recent Actions) preserved
- [ ] `--participants` CSV parser trims whitespace from each name before storing
- [ ] Existing entries in this repo with inconsistent participant names (mix of full/first name, leading-space variants, CSV artifacts) normalized to a single consistent form — one-time repo-local cleanup, not a framework convention
- [ ] `/sdd-catchup` sub-skill consumes enriched `sdd status` output rather than per-entry `sdd show` for kind + participants
- [ ] `/sdd-catchup` forwards kind and participants (direct + thread-level voices) in canonical structured form to the outer agent, not solely embedded in prose
- [ ] `/sdd-catchup` computes thread-level voices (full upstream chain, deduped names set) and surfaces them narratively only when they differentiate from the thread's active contributors
- [ ] `/sdd` Catch-up Playbook updated: item format unchanged in the common case; adds rules for thread-level participant mentions and optional active-recently header line
- [ ] `sdd lint` passes after cleanup
