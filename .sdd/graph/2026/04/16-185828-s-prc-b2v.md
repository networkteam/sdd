The /sdd skill instructs "Always use full entry IDs in --refs, --closes, and --supersedes flags" but the CLI rejects short suffixes for any ID argument, including positional ones like `sdd show <id>` and `sdd summarize <id>`. The flag-scoped wording slipped this session: full IDs were available in context from catch-up output but got shortened for `sdd show` calls, hitting "entry not found." The rule should be stated once, clearly, in a single place: full IDs when invoking the CLI; short IDs are fine (and easier to read) when presenting entries to the user. The same narrow framing appears in cli-reference.md's `sdd new` flags section.
