Establishes the GitHub Actions release pipeline infrastructure for SDD — `.goreleaser.yaml`, CI and release workflows, `.golangci.yml`, the `networkteam/homebrew-sdd` tap, and a binstaller-generated `install.sh` committed in-repo. First sub-plan implementing d-cpt-uu1; produces a working end-to-end release at v0.1.0 via Homebrew and a fallback curl installer. Skill extraction (`sdd install`), schema metadata, and `sdd migrate` remain deferred to a follow-on plan.

Deviates from d-cpt-uu1's attachment on two points, informed by 2026 landscape research:

- **Attestations**: native `actions/attest-build-provenance` + `gh attestation verify` replaces `actionutils/trusted-go-releaser`. GitHub's native attestations have become the idiomatic primitive; the wrapper adds indirection without trust-model benefit.
- **Curl installer**: retains binstaller-generated `install.sh`. `eget` was considered and rejected — it offloads bootstrap ("install eget first") rather than solving first-time install.

Supported platforms locked to darwin + linux × amd64 + arm64. CI test matrix mirrors release targets (ubuntu + macos); windows is neither built nor tested. Initial release tag is v0.1.0.

See [plan](./17-173310-d-tac-lg1/plan.md) for config scaffolds, resolved choices with reasoning, out-of-scope boundary, and rough sequencing.

## Acceptance criteria

- [ ] `.goreleaser.yaml` (v2 schema) builds darwin+linux × amd64+arm64 tarballs from `./cmd/sdd` with `CGO_ENABLED=0` and `-ldflags "-s -w -X main.version={{.Version}}"`.
- [ ] `main.version` declared in `cmd/sdd/main.go`; `sdd --version` prints the stamped version and exits 0.
- [ ] `.github/workflows/ci.yaml` runs on PR and push-to-main: `test` job matrix (ubuntu-latest, macos-latest) with Go 1.26 runs `go test ./...`; `lint` job on ubuntu-latest runs `golangci/golangci-lint-action@v8`.
- [ ] `.golangci.yml` present with enabled linters: errcheck, gosimple, govet, ineffassign, staticcheck, unused, gofmt, goimports, misspell, unparam; `go vet ./...` and `go test ./...` pass on main before the workflow lands.
- [ ] `.github/workflows/release.yaml` triggers on `v*` tag push: invokes GoReleaser v2 (`~> v2`), publishes GitHub Release with tarballs, checksums, and `install.sh`.
- [ ] Release workflow calls `actions/attest-build-provenance` across all release assets (tarballs, checksums, `install.sh`); `gh attestation verify --repo networkteam/sdd <asset>` succeeds on a published artifact.
- [ ] `networkteam/homebrew-sdd` tap repo created; `.goreleaser.yaml` `brews` block pushes formula on release using `HOMEBREW_TAP_TOKEN` secret (fine-grained PAT, contents:write on tap repo only); formula `test do` invokes `#{bin}/sdd --version`.
- [ ] `.config/binstaller.yml` initialized via `binst init --source=goreleaser`; generated `install.sh` committed to repo; CLAUDE.md documents manual regeneration when `.goreleaser.yaml` changes.
- [ ] README "Install" section documents three paths: `brew install networkteam/sdd/sdd`, basic `curl | sh`, verified `curl + gh attestation verify + sh`.
- [ ] v0.1.0 tag pushed; release workflow green; `brew install networkteam/sdd/sdd` succeeds on a fresh darwin machine; `install.sh` succeeds on a fresh linux machine; `gh attestation verify` succeeds on both the tarball and `install.sh`.
- [ ] Agent-neutrality preserved (d-stg-574): no Claude-specific logic introduced in the build pipeline, release workflow, or installer; skill handling remains deferred to the follow-on plan.
