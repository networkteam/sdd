The claude CLI exposes per-call cost and token metrics via --output-format json, but the llm.Runner currently discards this data, losing visibility into expensive entries and prompt optimization opportunities. The JSON response includes total_cost_usd, per-model input/output token counts, and cache hit/creation breakdowns. Surfacing these per-call stats would enable spotting expensive entries (large descriptions, many refs), tracking migration cost, and tuning prompt templates for token efficiency. Natural next step: extend llm.Runner to return structured metadata alongside the response, log per-entry stats during sdd summarize and sdd new pre-flight.
