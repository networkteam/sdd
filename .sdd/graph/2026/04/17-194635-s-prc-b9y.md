Rewrote README.md to cover framework concepts, installation, Claude skill setup, and `/sdd` session scenarios, and aligned the `sdd` binary path in all four skill files and their references from `framework/bin/sdd` to `bin/sdd` to match the actual dogfooding layout (commit fe5684a). The previous README only showed bare install plus `sdd new`/`sdd status` examples without explaining the skill-driven workflow — users had no hint that `/sdd` is the main entry point. The path references were holdovers from the pre-fork assumption that SDD would live at `framework/` in consumer projects; that convention doesn't hold in this repo and no formal install story exists yet (manual skill copy to `~/.claude/skills/` or project `.claude/skills/`, no `sdd install`). Graph entries under `.sdd/graph/` that still mention `framework/bin/sdd` were deliberately left untouched per immutability. Builds on the fork completion (a-tac-q94) as post-fork onboarding polish.
