Attachment referencing mechanism for graph entries.

Discovery: Attachments are discovered implicitly by scanning for a directory matching the entry's base filename. The graph model populates an attachments list from the directory contents. sdd show displays discovered attachments.

Referencing in entry body: Markdown links using {{attachments}}/filename placeholder syntax. The CLI resolves {{attachments}} to the actual relative directory path at creation time.

Immutability: Attachments are sealed on commit — immutable after the entry is captured. To update, create a new entry with --supersedes and fresh attachments.

Formats: Favor markdown (with mermaid for diagrams). Images allowed for design assets and screenshots.

CLI: sdd new accepts --attach path1,path2 flags. Single command, single atomic commit. The CLI generates the ID, resolves {{attachments}}, copies files, commits everything.

Example:

Agent prepares /tmp/design.md, then runs:

  sdd new d tac --attach /tmp/design.md "Decided on X. See [design notes]({{attachments}}/design.md) for trade-off analysis."

CLI generates ID 20260409-110000-d-tac-abc. On disk:

  graph/2026/04/09-110000-d-tac-abc.md        — entry
  graph/2026/04/09-110000-d-tac-abc/           — attachment dir
    design.md

Resolved link in entry: [design notes](./09-110000-d-tac-abc/design.md)
