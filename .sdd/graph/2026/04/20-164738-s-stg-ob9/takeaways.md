# Key Takeaways — Verbatim

Source: Simon Willison, "An AI state of the union: We've passed the inflection point and dark factories are coming," Lenny's Podcast
Video: https://www.youtube.com/watch?v=wc8FBhQtdsA

## Context is the competitive differentiator

> "It's hard to overstate how good these things are with reusing context that you can make available to them... Coding agents can do searches so you can give them access to an entire hard drive full of stuff and tell them what you need to solve and they will run search tools to find just the examples that they need to piece things together. It's incredibly powerful."

## The hoarding pattern

> "The way you build value as a software engineer or pretty much any other profession is you build a really big backlog of things that you've tried in the past that worked or didn't work such that when a new problem comes along you can think okay well in 2015 I built a system that used redis to do an activity inbox and then in 2017 I did rate limiting with node.js I can combine those two things right now and that will solve this new problem."

> "AI makes that so much easier because now I can knock out a very quick prototype that tries out this new NoSQL database or whatever it is cost me nothing to do. I've now got a markdown file somewhere with the output of the document."

## Feeding prior context to agents

> "I'll often just tell Claude code, here's pasted a URL to this thing, this thing here, here's another thing. Go and read the source code and then solve this new problem. And it works so so well."

> "I'll say things like check out simonwressearch from github and look at how look at the ones in there that deal with web assembly and rust and then use that to feed into solving this new task in web assembly and rust."
