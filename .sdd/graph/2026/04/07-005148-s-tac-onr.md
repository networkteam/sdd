Graph package has no way to construct a Graph in-memory for testing without the filesystem. All tests must go through LoadGraph with temp dirs and real .md files. This couples unit tests to file I/O and ParseEntry — can't test query logic in isolation. A builder or direct struct construction would enable faster, more focused tests.
