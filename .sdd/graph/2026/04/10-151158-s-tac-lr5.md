sdd new --attach takes file paths but doesn't support renaming the attachment in the target directory. Piping content via /dev/stdin or using temp files results in meaningless filenames (stdin, tmp.XXX). The CLI needs a filename mapping syntax (e.g. --attach source:target or --attach /dev/stdin:plan-requirements.md) so that skill usage and piped content produce descriptive attachment names without manual git mv cleanup.
