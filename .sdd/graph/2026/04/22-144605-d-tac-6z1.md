Extend pre-flight participant validation to consult config canonical and the full graph set as authoritative inputs via an LLM-judged check with explicit severity rubric, and surface the local canonical in `sdd status` output so agents have ground truth at check-in — closing the coverage gap named in s-prc-qom where the current exact-string validator was blind to same-person variants.

The two-sided structural fix addresses both failure modes per d-cpt-vt1: validator-side gets canonical ground truth as input (removes pollution via same-session drift in the reference set); agent-side has ground truth visible in the surface it already reads at check-in (prevents pattern-matching on drift-containing context). The validator check moves into the LLM layer because same-person variant detection (Christopher / Christopher Hlubek / Christopher H. / Chris Hlubek / Mr. Hlubek) is a semantic match no exact-string check can do — and the same channel handles narrative-introduced new voices cleanly. Severity is binary: pass (exact match or explicit new-voice introduction) or high-blocking. Full design dialogue, alternatives considered (including rejected layer-on-top approach), and open implementation questions in [plan.md](./22-144605-d-tac-6z1/plan.md).

## Acceptance criteria

- [ ] `PreflightQuery` gains a `LocalCanonical string` field populated from `.sdd/config.local.yaml`'s `participant` via a config-read finder, wired by the command handler (CQRS-aligned per d-cpt-ah1).
- [ ] Pre-flight uses an LLM-judged participant check taking `LocalCanonical`, `EstablishedParticipants`, and entry content; emits exactly one high-severity `participant-drift` finding per unmatched participant unless the description explicitly introduces them as a new voice joining the graph. Bootstrap exception: empty `EstablishedParticipants` → no finding.
- [ ] Template calibration includes three exemplars: the Christopher variant chain (high), a qualifying new-voice introduction (pass), a non-qualifying mention like "X attended the meeting" (high).
- [ ] Existing deterministic `participantDriftFindings` in `internal/finders/preflight.go` is removed — LLM check is the single path.
- [ ] `sdd status` output adds a `Local participant: <canonical>` header line above the `Graph:` summary; shows `(not configured — run sdd init)` when config is missing.
- [ ] `/sdd` SKILL.md names the status header as the canonical source for composing `--participants` lists and reflects the stricter rubric. Skill source in `internal/bundledskills/claude/sdd/` + installed copy rebuilt into `.claude/skills/`.
- [ ] Regression tests cover: exact match passes, canonical variant blocks high, novel unmatched blocks high, explicit new-voice introduction passes, non-qualifying mention blocks high, bootstrap (empty graph) passes, multi-participant entry with one drift yields exactly one finding.
