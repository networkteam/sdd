Cross-model testing with gemma4:26b and Haiku-via-API confirms the summary template's regressions are template-level, not model-specific. gemma4:26b shows the most severe drift (collapsed enumerations, lost concrete examples, generic "This signal identifies..." openers, self-referenced IDs); Haiku via the Anthropic API produces quality comparable to or slightly better than the Claude Code session path but still occasionally slips into the self-ID reference pattern and generic opener. Both exhibit the self-ID reference bug — strongly implies the template needs an explicit prohibition against self-reference plus a concrete first-sentence exemplar.

Separately, found a gollm quirk: default `MaxTokens=256` silently truncated longer Haiku summaries mid-sentence until set to 4096 in the adapter — worth remembering when onboarding other gollm-backed providers.

Supersedes s-cpt-nys which captured the same thesis from gemma4 alone; cross-model confirmation sharpens it.
