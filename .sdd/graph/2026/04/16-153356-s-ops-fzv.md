Pre-flight and summary generation in sdd new run sequentially (handler_new_entry.go:82-135), adding 30-60s of unnecessary wall time per entry. Both are independent LLM calls that can run concurrently. The summary is wasted if pre-flight rejects, but that is a cheap Haiku call vs. saving 30-60s on every accepted entry — most entries pass. Implementation: launch both as goroutines after validation, wait for both, discard summary if pre-flight blocks.
