`sdd summarize --all` is fully sequential with no prompt caching, making bulk re-summaries slow at scale — each entry blocks on a 30-second `claude -p` round-trip. This motivates a pluggable provider abstraction where `claude -p` stays default (logged-in Claude Code session, no API key) but can be swapped per-machine for local Ollama (fast, free, no rate limit) or the Anthropic API via a unified client library (enables ephemeral prompt caching on the stable template prefix). The same abstraction opens the door to errgroup-pooled concurrency for batch summarize. The pain surfaces specifically during catch-up work where many changed entries need re-summarization in one pass.
