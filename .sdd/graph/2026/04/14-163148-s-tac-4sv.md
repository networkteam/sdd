Built the summary generation infrastructure: llm package, sdd summarize command, and lint integration. New sdd/llm package owns all LLM generation concerns — preflight moved from finders for consistency, summary generation added alongside it. Each entry gets a summary field (2-3 sentences describing entry and direct relationships) and summary_hash (SHA-256 of the full rendered prompt, so template, content, or upstream summary changes automatically invalidate downstream hashes). The sdd summarize command processes entries in DAG topological order, skips entries whose hash matches, and commits in one batch. Lint detects missing or stale summary hashes. Migration (sdd summarize --all) is in progress. Remaining from the plan: generation at sdd new time, CLI output changes (list/show/status consuming summaries), and skill updates (catchup/explore using summaries for closed entries).
