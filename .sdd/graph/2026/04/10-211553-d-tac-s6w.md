sdd new invokes claude -p as a pre-flight validator before creating graph entries. The CLI selects the appropriate pre-flight check based on operation type (closing action, decision, signal, supersedes). Pre-flight prompt templates are embedded via go:embed into the sdd binary. The validator sees the proposed entry plus relevant graph context (referenced entries, plan items, active contracts). Hard gate — entry creation is rejected with reported gaps on failure. Moves the pre-flight checklists from d-prc-4du out of the prompt/instructional layer into the tool layer. See attachment for full design: operation flow, checks per type, contract enforcement, implementation details, CLI flags.
