Closes s-prc-sbl by restructuring the `/sdd` catch-up flow so mechanical graph state never routes through LLM synthesis. Root cause: the `/sdd-catchup` sub-skill was asked to emit a per-entry `Status` field, which it synthesized from summary prose — producing contradictions like "Active plan (closed/implemented)" for entries the CLI had already filtered to the active set. Section membership in `sdd status` is the status; no synthesis needed.

Fix spans five commits on `future-framework`:

- `4767538` — drop `/sdd-catchup` sub-skill from catch-up; outer agent runs `./framework/bin/sdd status` + `sdd wip list` directly; explicit rule: no per-entry `Status` label.
- `b670e17` — rewrite Participants Playbook for direct `sdd status` reads (collapse thread/item rules into "outside voices only", remove user-specific example names).
- `80599b6` — add strict completeness: every entry under Plans, Active Decisions, Open Signals must appear with its own number.
- `32fe423` — exclude contracts from the numbered list (standing context, not action items).
- `0b450a5` — drop redundant "count items before finishing" instruction.

Validated across three fresh `/sdd` sessions: no fabricated lifecycle labels, no silent drops, mechanical completeness holds.

Broader principle named in dialogue but not captured as a separate entry: deterministic graph state does not belong in LLM-synthesis paths. Same pattern as s-prc-hpa (deterministic field checks in pre-flight) and s-prc-v69 (kind/participants missing from CLI output) — all three resolve by moving mechanical work out of LLM layers. Worth capturing as its own higher-altitude signal if the pattern surfaces again.
