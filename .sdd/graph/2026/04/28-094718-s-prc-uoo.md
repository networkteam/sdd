Agent-authored narration toward people drifts from plain-language process description into SDD-internal mechanics in two ways: jargon-heavy register, and reproduction of playbook moves that already live in the base skill. Observed when generating a workshop meeting agenda in a German-language SDD project: the agent appended a "Nach dem Meeting — Rückführung in den Graph" section that used SDD wording (done-signals, supersede chains, kind names) and re-described the agent's own back-fill procedure step-by-step — content belonging to the skill's behavior, not to a meeting agenda. The same register issue appeared during the bootstrap onboarding evaluation (s-prc-yg0). The remedy is positive, not suppressive: outer narration toward people uses plain language to describe the process with clear words, while precise entry IDs for referencing and brief connecting details to the graph remain fine; and generated artifacts must not duplicate playbook moves already encoded in the base skill — the agent operates the process, the artifact narrates the substance.

See [evidence](./28-094718-s-prc-uoo/evidence.md) for the verbatim excerpt, the two failure modes, and a before/after sketch.
