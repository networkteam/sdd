SDD's GitHub-facing positioning should lead with outcome — "Keep humans and AI agents aligned across parallel work" — and contrast SDD against two distinct failure modes: agent sessions where insights and decisions get lost once the session ends, and traditional systems that scatter state across separate static artifacts (docs, plans, phases, roadmaps, trackers, specs, tickets) that duplicate across layers, go stale, and resist new insight. SDD's append-only graph with derived state is the mechanism that answers both. Audience umbrella is "agentic software teams and solo developers" — neutral between solo and collaborative contexts, avoiding PM-replacement framing that creates resistance. The current README opens with abstract framework language ("a framework for building and traversing decision graphs through human-agent dialogue") that buries pain, audience, and outcome. This signal reframes how SDD presents itself publicly and will drive a tactical plan decision for a README rewrite. Sources: external 2026 developer-tool marketing advice plus dogfooding observation on multi-threaded work productivity.
