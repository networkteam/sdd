Add short-ID acceptance to the `sdd` CLI. Any argument that currently requires a full entry ID (positional args for `sdd show`, `sdd summarize`; flags `--refs`, `--closes`, `--supersedes` on `sdd new`) will also accept the short form `{type}-{layer}-{suffix}` (e.g. `s-prc-oka`, `d-cpt-omm`) when it uniquely matches one entry. On ambiguity, the CLI always exits non-zero and lists all matching full IDs as alternatives — no fallback behavior. Not-found errors are unchanged.

Motivation: s-prc-b2v surfaced a UX trap where short IDs are natural for humans copying from `sdd status` output but the CLI rejects them. The split: agents continue to use full IDs (deterministic, collision-proof as the graph grows — enforced in the /sdd skill); the CLI is forgiving enough that direct user invocation does not require typing or looking up a timestamp. `sdd new` write flags rarely get called by users directly, but universal scope is simpler than carving out exceptions.

Alternatives considered: suffix-only matching (e.g. `oka`) — rejected as too collision-prone; path-argument-only acceptance — rejected as inconsistent with `--refs`/`--closes`; silent disambiguation via most-recent or first-match — rejected to preserve deterministic behavior and surface real collisions to the caller.

## Acceptance criteria
- [ ] CLI resolves short-form IDs of shape `{type}-{layer}-{suffix}` against existing entries; unique matches resolve to the full ID transparently
- [ ] Ambiguous short-ID input always exits non-zero and prints all matching full IDs as alternatives — no fallback behavior (no "most recent wins", no "first match wins")
- [ ] Short IDs that match zero entries exit non-zero with the same not-found error as full IDs that do not exist
- [ ] Full-ID inputs continue to work exactly as before — no behavior change for fully-qualified calls
- [ ] Short-ID acceptance applies to: `sdd show <id>`, `sdd summarize <id>`, and the `--refs`, `--closes`, `--supersedes` flags on `sdd new`
- [ ] The /sdd skill text keeps its rule that agents invoke the CLI with full IDs (updated to be stated in a single clear place, covering all ID arguments, not just three flags)
