Real-git smoke testing of the background sync feature (d-tac-hsu, closed by s-tac-2y0) surfaced two bugs that the mock-based unit tests had missed, reinforcing that code at the boundary against external tools needs exercise against the real tool even when domain logic is well-covered. Two specific drifts from the plan ACs were found and corrected:

First: `git merge-tree --merge-base=<oid>` was introduced in git 2.40 (Dec 2022), not 2.38 as the plan's implementation notes assumed — macOS ships git 2.39.5 (Apple Git-154) which rejects the flag with "unknown option". Fix drops the explicit flag and relies on git computing the merge base internally from the two branch args; behavior is functionally identical. The plan AC5 and the closing done signal both name the `--merge-base=<base>` form verbatim, so there is a textual drift from the graph record, but the conflict-prediction outcome is unchanged.

Second: `git merge-tree --write-tree --name-only` emits trailing informational messages ("Auto-merging <path>", "CONFLICT (content): ...") on stdout by default, separated from the path list by a single blank line. The production parser was treating those messages as additional conflict paths, leading to warn lines like "sync: rebase would conflict in shared.txt, Auto-merging shared.txt, CONFLICT ...". Fix adds `--no-messages` to the production invocation and tightens `ParseMergeTreeConflicts` to stop at the first blank line as defense-in-depth.

Both bugs sit in the thin layer between `gitSyncerImpl` and the real git binary — the exact surface mocks cannot exercise. Unit tests with `fakeGitSyncer` validate the finder's state-routing logic correctly (and caught multiple edge cases during development), but they cannot catch "the command we think we're running is not the command git accepts" or "the output we think git produces is not the output git produces". The process takeaway: when a new SDD feature shells out to an external tool, an end-to-end smoke test against a throwaway real setup is load-bearing, not optional. Five states were verified (up-to-date, fast-forward, clean-rebase, conflict-predicted, local-ahead) plus cooldown silencing; fix committed in ef2b057 on main.
