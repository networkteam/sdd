Participant identity is the responsibility of the outer skill (e.g. /sdd), not the CLI. The skill infers participants from session context — git config for the user, conversation context for whether it's a group session. Since all captured entries are presented to the user for confirmation before sdd new runs, misidentification is correctable. The CLI accepts --participant flags as given and has no identity mechanism of its own.
