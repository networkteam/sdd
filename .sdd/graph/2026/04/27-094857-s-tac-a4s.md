Shipped per-model rate-limit defaults for remote LLM providers (commit `80d2d3e`) so `sdd summarize --all` and other batch operations no longer trip 429s at Anthropic/OpenAI tier 1 without manual configuration. When `llm.rate_limit_rps` is unset, the factory applies a conservative tier-1-safe cap based on provider/model family: Anthropic Opus 0.5 RPS, Sonnet 1.0, Haiku 2.0; OpenAI frontier 1.0, mini/nano 2.0 — grounded in Anthropic's family-shared ceilings (50/100/200 RPM) and OpenAI per-family documentation. Users on higher tiers override via `llm.rate_limit_rps` in `.sdd/config.local.yaml`.

The gap raised three concerns: config field, provider-specific defaults, automatic 429 backoff. The config field had already landed in commits `33bf30f` + `1721da5` before the gap was filed. This change delivers provider-specific defaults. Automatic 429 backoff is deferred: dialogue surfaced that gollm's built-in retry is deliberately disabled at `gollm.go:49` to avoid batch-log flooding on cancelled contexts, so proper 429 handling would need a bespoke retry layer that classifies errors and respects `retry-after` headers. The conservative defaults should prevent 429s in steady state; if bursty traffic becomes felt friction, we add a scoped retry layer then rather than pre-building one.
