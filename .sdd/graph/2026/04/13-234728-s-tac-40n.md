Completed the CQRS migration per d-tac-lce / d-tac-lwm — see commit 10aae02. All remaining commands and queries (`lint`, `status`, `show`, `list`, `wip list`, `wip start`, `wip done`) now flow through command→handler or query→finder + presenter pipelines; the old top-level `framework/sdd` package is gone.

Open questions in the plan resolved during session-start dialogue: `presenters/` created as a dedicated view layer (overriding the plan's lean toward CLI-owned presentation — view-layer separation enables independent testability); `LoadGraph` and `LoadWIPMarkers` moved onto the `Finder`; `framework/sdd/` deleted entirely. Handler now takes a single `Reader` interface satisfied by `*finders.Finder`. Added a `Brancher` interface for git branch ops (`gitBrancher` is the production impl).

Test disposition (per d-tac-lwm step 7): `graph_test.go` and `wip_test.go` moved from `sdd/` to `finders/` (now exercise the Finder methods). `render_test.go` moved to `presenters/show_test.go`, rewritten as integration tests through `Finder.Show` + `presenters.RenderShow` (same fixtures, same assertions on rendered output). `test_helpers_test.go` duplicated into `finders/` and `presenters/` since each test package needs its own copy of the entry-builder helper. Existing `handler_new_entry_test.go` updated to use the new `Reader` interface via a combined `fakeReader` stub. The dry-run regression tests for d-prc-31v carry forward unchanged.

No new dedicated unit tests added for the new finders (Lint, Status, List, WIPList) or new handlers (StartWIP, FinishWIP). `Finder.Show` is exercised indirectly through the presenter integration tests; the others are thin pass-throughs to existing model methods (Status / List / Lint) or wrap LoadWIPMarkers (WIPList). The wip start/done handlers were validated by manual smoke testing; this is a refactor (existing CLI behavior preserved), not a bug fix, so d-prc-31v doesn't apply. New tests for these handlers can be added when behavior changes or bugs surface.

Dead code stripped from `cmd/sdd/main.go` (per d-tac-lwm step 8): the `printEntry`, `groupByLayer`, `layerOrder`, and `deleteBranch` helpers were removed once the last consumer migrated to a presenter or to `gitBrancher`. `go vet ./...` reports clean; only the production CLI plumbing remains in main.go.

Bundled into one commit at the working-state checkpoint instead of one per command per the plan's suggestion.
