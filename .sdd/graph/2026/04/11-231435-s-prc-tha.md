Claude Code sessions are bound to their project directory. When sdd wip start --branch creates a sibling worktree, the user must manually start a new agent session there — the creating session cannot continue in the worktree. Graph context transfers fine via git, but session setup is not automatic. For now, the default should be branching without worktrees (just git branch, no concurrent work on the same repo directory on the same system). Worktree creation should be an explicit opt-in for users who need true concurrent work and accept the session transition overhead. This keeps the common case simple and defers the tooling friction to when it's actually needed.
