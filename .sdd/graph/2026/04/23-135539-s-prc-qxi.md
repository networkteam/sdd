Multilingual infrastructure (d-cpt-x38) shipped without user-facing discovery — `sdd init` doesn't prompt for `language` and the README makes no mention, so users who don't read `.sdd/config.yaml` comments never learn the feature exists. Fix has two parts: prompt for language during `sdd init` with `en` as default (parallel to existing participant / graph-dir prompts, writing the chosen value explicitly so "did I configure this?" doesn't arise) and add a dedicated multilingual subsection to README explaining what enabling a locale does, which locales are bundled, and what stays English regardless.
