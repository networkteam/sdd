Add background sync awareness to the SDD CLI and skill. Depends on .sdd/ metadata infrastructure (d-tac-s2g) for config and tmp storage. On any sdd command, auto-fetch with a configurable cooldown (from .sdd/config.yaml), then compare local vs remote by scanning commit log messages for graph entry patterns — no file-level inspection needed. Report both directions: entries to pull and entries to push. The SDD skill acts on these notifications: pull --rebase when git status is clean and remote is ahead; suggest push when local is ahead.

## Acceptance criteria
- [ ] sdd commands run git fetch when last fetch exceeds cooldown from config; timestamp persisted in .sdd/tmp/last-fetch
- [ ] After fetch, CLI detects remote-ahead graph commits by commit message pattern and emits a log line with count
- [ ] CLI detects local-ahead graph commits and emits a log line with count
- [ ] Git command count minimal: one fetch per cooldown window, log comparison via git log range, no file diffs
- [ ] SDD skill updated: pulls with --rebase when status is clean and remote is ahead; suggests push when local is ahead
