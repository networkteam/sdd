Persist stdin attachment content on pre-flight rejection so retries don't require re-sending. When sdd new receives attachment via -:filename.md and pre-flight rejects the entry: (1) write the stdin content to .sdd-tmp/<hash>-<target-filename> in the repo root, (2) output the saved path in the rejection message, (3) on next successful sdd new, clean up .sdd-tmp/. Add .sdd-tmp/ to .gitignore.
