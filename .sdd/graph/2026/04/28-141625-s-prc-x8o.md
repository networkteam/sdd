Agents working with the SDD skill infer approval-to-capture from intermediate dialogue signals — answered questions, addressed concerns, integrated design points — rather than waiting for an explicit go signal from the user. Observed twice in close succession: in this project, the agent captured a tactical plan immediately after rolling a newly-raised version-skew concern into the ACs; in an external project, the same approval-by-inference pattern recurred after one or two answered questions. Auto-mode framing in the harness ("minimize interruptions", "prefer action over planning") amplifies the failure mode by pushing against SDD's playback-before-capture discipline.

Per d-cpt-vt1, instructions alone are unreliable for behaviors of this kind, and a clean structural gate would be ideal. But the available structural mechanisms here all have material costs or limits in the current Claude Code model: a two-step `sdd draft` → `sdd commit` flow introduces drag on every capture without structurally preventing the agent from running both steps; confirmation tokens or hash bindings cannot bind to the user since both playback and capture flow through the agent; an interactive stdin gate cannot reach a real user tty from the agent's subprocess; harness permission prompts work but are user-configured and erode under fluency pressure; a pre-flight extension that scans dialogue for explicit approval cannot actually access the conversation transcript. Pre-flight already provides a content-level structural gate — it caught an earlier instruction-shaped capture of this very signal — but a temporal gate around dialogue approval is genuinely hard given the agent-shell interface.

The pragmatic step this signal proposes is a SKILL.md refinement that names the rule plainly: `sdd new` requires an explicit user signal ("yes, capture", "go ahead", "ready"), and resolved questions, addressed concerns, and integrated design points never substitute on their own. This is acknowledged as instructional reinforcement rather than a contract-pure structural fix; the gap remains open for a future structural shape (cheap-revert framing, harness permissions, or an external transcript-reviewing sub-agent) when one becomes feasible. The recursive observation that pre-flight caught the prose-only first draft of this signal is itself evidence the contract's structural-enforcement principle is doing partial work today, even if a temporal gate is not yet in reach.
