The `sdd summarize --all` command with the default concurrency immediately exceeds Anthropic's rate limits (50 RPM and 50K input tokens/min on `haiku-4-5`), requiring manual `--concurrency 1` which took 6:42 for ~30 stale entries during session 5 of the migration. Provider-aware rate-limit handling — automatic backoff on 429, provider-specific concurrency defaults, or a rate-limit config field in `.sdd/config.yaml` — would prevent this paper cut from recurring on every bulk summarize. Complements `s-cpt-ed2`'s secondary observation on gollm's MaxTokens default: both are provider-onboarding defaults that bite on first bulk use, and together suggest a broader pattern of provider defaults that need explicit calibration before the pluggable layer (`s-tac-i8f`) is fully ergonomic.
