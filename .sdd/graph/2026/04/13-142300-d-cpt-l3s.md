Adopt CQRS as the architectural pattern for all framework Go code. CLI actions become thin shells that build command/query structs and dispatch them to handlers/finders; the existing `sdd/` library primitives (Entry, Graph, parsing) stay as building blocks composed by the new layer.

**Package layout** under `framework/sdd/`:
- `command/` — domain command structs (e.g. `NewEntryCmd`). Each carries fields capturing intent and optional callback functions for results (e.g. `OnNewEntry func(id string)` — just the new entry's identifier). Has `Validate()` for basic required-fields checks; richer validation is the handler's job.
- `query/` — domain query structs. Pure read intent.
- `handlers/` — one `Handler` struct per area (not per command, to avoid boilerplate). Methods per command (e.g. `Handler.NewEntry(ctx, cmd)`), one method per file (`handler_new_entry.go`). Holds injected dependencies (graph dir, git committer, pre-flight runner, clock, stdin reader).
- `finders/` — process query structs into results. Pure reads, no side effects. May be used by handlers and called directly from CLI.
- `model/` — pure domain types and logic shared across the layer. No IO, no side effects.

**Architectural rules** (the rigor):
- **Side effects only in handlers.** Model and finders are pure. Everything outside handlers becomes trivially testable.
- **Handlers return only errors.** Small result identifiers flow back through callbacks on the command struct (e.g. \`cmd.OnNewEntry(id)\` — just the ID, not the full entry). If the caller needs richer data, it issues a query against the appropriate finder after the handler completes. This keeps handlers from becoming a back-door for queries — write paths and read paths stay distinct, and "I'll just have the handler also return X" stops being an option.
- **Handlers may use finders** internally and reach into model/library primitives within their bounded context.
- **Pre-flight is a query** (pure read; the LLM call is a side effect of the runner dependency, not of the gate's intent), so it lives in \`query/\` + \`finders/\` rather than \`handlers/\`.

Builds on s-prc-17y (regression-test gap surfaced testability problem with the current Action-closure structure) and enables d-prc-31v (the regression-test contract — handlers with injected deps make tests trivial). First concrete application is the \`sdd new\` refactor; the rest of the migration follows mechanically once the reference implementation is reviewed.
