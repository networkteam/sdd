Background sync awareness implemented per d-tac-hsu. Running any sdd command except `sdd init` now auto-fetches when the last-fetch marker exceeds a configurable cooldown (default 15m from `.sdd/config.yaml`), detects both remote-ahead and local-ahead graph commits by the `^sdd:` commit-message pattern, and emits a single slog line summarizing the state. When both sides have diverged, the CLI consults `git merge-tree` to predict rebase safety and distinguishes clean-rebase from conflict-predicted states. The /sdd skill SKILL.md documents the contract so the skill can pattern-match each `sync:` line and act — auto-pull `--rebase` (tell-then-run) on safe states with a clean working tree, warn with conflicting paths on unsafe states, suggest push when local is ahead, and surface setup/network warnings without auto-fixing. Verified end-to-end on this feature branch (no upstream → warning fires once, cooldown silences subsequent invocations), full test suite green, golangci-lint clean.

Acceptance criteria:

- [x] sdd commands except `sdd init` run `git fetch` when last-fetch exceeds cooldown from config; default `15m`; marker persisted at `.sdd/tmp/last-fetch`. Wired via the root `Before` hook in `cmd/sdd/main.go`; init is skipped by matching `cmd.Args().First() == "init"`. Config key is `sync.cooldown` resolved by `model.ResolveSyncCooldown` with graceful fallback to the default when empty or unparseable.
- [x] CLI detects remote-ahead graph commits by commit-message pattern and emits a slog line with count. Production `gitSyncerImpl.CountCommits` runs `git log --grep='^sdd:' -E --pretty=format:%H HEAD..@{u}` and the finder counts non-empty lines; emission format is `sync: fast-forward available, N commits behind` (info).
- [x] CLI detects local-ahead graph commits and emits a slog line with count. Same plumbing over `@{u}..HEAD`; emission format is `sync: local ahead by N, consider push` (info).
- [x] Git command count minimal: one fetch per cooldown window (last-fetch gate), one `git log` per direction (fast-forward / local-ahead / up-to-date states), plus `git merge-base` + `git merge-tree` only when both sides diverge. No file-level inspection anywhere.
- [x] CLI predicts rebase conflict safety when local and remote have diverged: runs `git merge-tree --write-tree --name-only --merge-base=<base> HEAD @{u}`; parser in `model.ParseMergeTreeConflicts` extracts conflict paths from stdout after the tree OID; exit code 1 is treated as the documented conflict signal. Distinct states emitted: `sync: rebase is clean, N remote / M local` (info) or `sync: rebase would conflict in <paths>, N remote / M local` (warn).
- [x] SDD skill updated to auto-pull `--rebase` (tell-then-run) only on clean working tree + (fast-forward OR clean-rebase-predicted); on conflict-predicted or dirty working tree, warns with conflicting paths and defers; suggests push when local is ahead. New section "Reacting to sync-check output" in `internal/bundledskills/claude/sdd/SKILL.md` — installed copy refreshed via `sdd init --scope project` and auto-committed.
- [x] Missing git setup (not a repo / no remote / no upstream) and transient fetch failures emit `slog.Warn` at most once per cooldown window; commands proceed without error. The finder touches the last-fetch marker on terminal failure states (no-remote, no-upstream, fetch-failed) so the cooldown bounds re-attempts to once per window.
