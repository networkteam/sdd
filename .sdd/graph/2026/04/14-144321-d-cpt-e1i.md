The SDD framework uses an immutable decision graph implemented on Git. Entry information — the semantic content and authorial intent — does not change after creation. A decision is never edited into a different direction; it is superseded by a new decision. Current state is always reconstructed by traversing the graph, never maintained separately. Each fact lives in exactly one place.

Mechanical fixes to entry files are permitted: adding missing metadata fields, correcting links that were missed during capture, fixing formatting. These change the file but not the information. The test: would a reader's understanding of the entry change? If yes, supersede. If no, fix.
