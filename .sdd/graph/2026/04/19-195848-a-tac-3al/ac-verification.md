## Dialogue deviations

**Source-of-truth location**: Plan's open question "`//go:embed skills/*/**` vs `//go:embed .claude/skills/*/**`" settled mid-session — skills live at `internal/bundledskills/claude/` (source) and `.claude/skills/` became the installed output. Go's `//go:embed` constraint forced this (can't reach files outside the package subtree). User confirmed Option A ("Invert: source of truth moves to `internal/bundledskills/skills/`") and explicitly flagged that `.claude/skills/` should not be part of SDD's documented structure — which shaped the README rewrite.

**Live-skill preservation during implementation**: User requested the physical `.claude/skills/` files stay in place during this session (active skill use), so the embed source was copied rather than moved. CLAUDE.md now documents the edit-in-internal / restamp-via-init workflow.

**`--force` flag**: Not in the original plan, added mid-implementation at user request for non-interactive overwrite. `InstallSkillsCmd.Force` short-circuits the prompt path in the handler; CLI exposes it as `--force` on `sdd init`.

**Unstamped-matching file rule**: Added to `ComputeSkillStatus` during dogfood smoke-testing — files predating the stamping era (no `sdd-content-hash`) whose body matches the embedded bundle classify as Pristine so first-time adoption refreshes silently instead of prompting. Covered by `TestComputeSkillStatus_UnstampedMatchingIsPristine` and `TestComputeSkillStatus_UnstampedDifferentIsModified`.
