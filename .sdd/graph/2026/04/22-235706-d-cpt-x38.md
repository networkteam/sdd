Multilingual SDD uses a per-graph configured language in `.sdd/config.yaml` (top-level `language`), with captured entries authored in that language while conversational dialogue flows freely in any language. CLI tokens (type, kind, layer names) remain English as canonical identifiers — mechanical validation and the type-system contract (d-cpt-ydf) key on them — and the `/sdd` skill renders translated vocabulary to users via bundled translation references per locale, reading the relevant vocabulary file on demand. The configured language is surfaced in `sdd status` output (parallel to the `Local participant:` header) so the agent has it at session start without an extra config call. The skill canonicalizes at capture time, translating dialogue content into the configured language before entries are written, and pre-flight flags language drift parallel to the participant-drift check. CLI output (`sdd status` headers, `list`, `show` body) stays English; skill-layer translation is the single localization surface during dialogue. Closes the multilingual question (s-cpt-y33). See [plan](./22-235706-d-cpt-x38/plan.md) for rationale, mechanics, and alternatives considered.

## Acceptance criteria

- [ ] `.sdd/config.yaml` supports top-level `language` (locale code); default is English when unset
- [ ] `sdd status` surfaces the configured language in its header so the agent has it at session start without an extra call
- [ ] A non-English vocabulary reference exists at `internal/bundledskills/claude/sdd/references/vocabulary-<locale>.md`, covering type, kind, and layer names; German (`de`) is the first case
- [ ] `/sdd` skill instructions direct the agent to read the locale-specific vocabulary reference on demand and use translated terms in user-facing rendering (catch-up, playback, status narration, grooming tables) when `language` is non-English
- [ ] `/sdd` skill guidance specifies capture-time canonicalization — dialogue in other languages is translated into the configured language before `sdd new`
- [ ] Pre-flight detects language drift: an entry whose description language does not match the configured language surfaces a finding (severity modeled on participant drift)
- [ ] End-to-end smoke: a session with `language: de` captures a German-language entry that passes pre-flight without `--skip-preflight`
