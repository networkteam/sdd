Search across graph entries relies on grep, which misses conceptually related entries when wording differs — the original semantic-search gap (s-tac-n2v) named this at ~80 entries and the closing done signal (s-tac-2y8) deferred resolution with the explicit condition "revisit when the graph grows by a magnitude." The graph now sits at ~395 entries (~5x growth) and friction is empirically visible: agents grep for keywords, miss related signals, and exploration leans on the LLM-as-semantic-matcher workaround inside sub-skills, which is expensive and unreliable as content diversifies. The friction will sharpen as multi-participant work scales — parallel sessions can capture signals on the same concept without sharing refs, and there's currently no mechanism to detect those silent divergences (per the alignment aspiration d-stg-qlt).

One initial sketch worth thinking through: an `sdd search` CLI command backed by vector embeddings of entry text, returning ranked top-N matches that the agent expands via existing graph traversal (`sdd show --max-depth`, `sdd list`) to surface **ranked clusters** of related entries rather than flat results — embeddings narrow, the graph's typed edges enrich. Two concrete application surfaces: **explore** finds connections the agent would miss with grep, and **grooming** detects clusters that should be related but aren't (under-linked neighborhoods, parallel captures of the same concept) — directly strengthening the existing sub-skills. Framing constraint from the dialogue-first aspiration (d-stg-beb): search is a tool the agent uses to find meaningful connections to walk through, not to generate the final answer. Open trade-offs include embedding model and store choice (local vs hosted; language portability now that German is in scope), index-regeneration cadence on capture, drift between description and summary content, and build-vs-leverage given that editors increasingly ship semantic search natively. Adjacent discovery gaps on tag-based clustering (s-cpt-s43) and external/session conversation mining (s-cpt-bn1, s-cpt-5ox) may share embedding infrastructure if pursued.

Reopens the deferred semantic-search question (s-tac-n2v / s-tac-2y8) with empirical signal of friction at sub-magnitude scale.
